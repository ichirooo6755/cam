
## 日中撮影の取扱説明

### 白飛びする理由

```
フィルムカメラのレンズ → 明るい屋外の光を集光 → Piセンサーに直撃
Piセンサーのgain最小値 = 1.0（ISO100）
それでも飽和する = レンズが集める光が多すぎる
```

### 日中の撮影方法

**フィルムカメラ側のレンズ絞りを上げる（f/8〜f/16）。** これが唯一確実な対策。Piセンサー側では光量を減らせない。

| 条件 | 推奨絞り |
|------|---------|
| 快晴屋外 | f/11〜f/16 |
| 曇り屋外 | f/5.6〜f/8 |
| 室内 | f/2.8〜f/4 |
| 夜 | f/1.4〜f/2.8（適応型gainが自動で上がる） |

### SSは早くできないの？

**できるが、短くすると別の問題が起きる：**

```
SS 1秒（現在）: 1fpsで確実に光を捕捉。ただしフレーム間の待ちが最大1秒
SS 33ms（30fps）: 高速だが、ISPのブラックレベル補正で写真が真っ黒になる（実測済み）
SS 100ms〜500ms: 中間だが同様に黒潰れの可能性あり（要テスト）
```

Pi HQカメラ（IMX477）のISPが、暗いフレームのmainストリーム出力を0にクランプする仕様のため、短いSSでは光が入っても写らない。**現状SS1秒が唯一動作確認済みの値。**

### 今はautoが一番いい？

**現在の最適設定：**

| 設定 | 値 | 理由 |
|------|-----|------|
| `iso` | **auto** | 適応型gainが撮影結果から自動調整（ISO100〜1600） |
| `shutter_speed` | **1000000**（1秒） | 確実に光を捕捉できる唯一の確認済み値 |
| `camera_mode` | **standard** | バランス型 |
| `raw_mode` | **false** | trueだとDNG保存に失敗する場合がある |

ユーザーが手動でISOを設定した場合はそれが最優先。autoの場合だけ適応型gainが動く。

### 操作の流れ

```
1. フィルムカメラのレンズ絞りを調整（日中はf/8以上）
2. シャッターを切る
3. Piが光を検知→同一フレームを保存（遅延ゼロ）
4. 撮影後、適応型gainが次の撮影のISOを自動調整
5. 2枚目以降はより適正な露出になる
```

**1枚目は白飛び/黒潰れする可能性がある**が、2枚目以降は適応型gainが調整するので改善する。ただしgain=1.0が下限なので、明るすぎる場合は絞りで対処。
