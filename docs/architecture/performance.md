# 性能・省電力方針

## 目的
Pi Zero 2W の制約下で、撮影安定性とバッテリー持続時間を両立する。

## 基本方針
- Pi 側は「検知・撮影・保存」に集中
- 重い画像処理（合成、サムネ生成、装飾）は iOS 側へ寄せる
- API は軽量レスポンスとストリーム配信を優先する

## 高優先で効く改善
1. 多重露光や加工処理を iOS 側へ移す
2. 撮影ループの不要な待機や再設定を削減する
3. サムネイル取得のキャッシュと同時接続制御を導入する

## 期待効果（要旨）
- Pi 側 CPU スパイクの低減
- メモリ逼迫の緩和
- 撮影までの遅延短縮
- 消費電力低減

## 写真編集学習ロジック改善
- ヒストグラム分析（輝度分布、クリッピング）を追加
- 露出スコアで明暗バランスを評価
- 高評価データに加えて低評価データも学習対象に追加
- 学習データ量に応じて重みを動的に調整
- 類似画像優先で学習データを抽出し、処理時間増加とのバランスを取る
