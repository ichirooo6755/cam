# Pi Zero 2W 省電力最適化レポート

## 現状分析

### Raspberry Pi Zero 2W スペック
- CPU: Quad-core ARM Cortex-A53 @ 1GHz
- RAM: 512MB
- 消費電力: 0.4W (アイドル) ～ 1.3W (負荷時)
- **制約**: メモリ少、CPU性能控えめ → 画像処理は苦手

### PiCamera HQ (Sony IMX477) 対応状況
✅ **完全対応済み:**
- ネイティブ解像度: 4056x3040 (12.3MP)
- RAW（DNG）撮影: format='dng'
- ISO範囲: 100-16000 (推奨100-6400)
- 長時間露光: 13μs - 670秒 (推奨100μs - 1s)
- ノイズ除去: off/auto/cdn_off/cdn_fast/cdn_hq
- シャープネス: 0.0-16.0
- 手ぶれ補正（電子式）

## 最適化戦略

### Phase 1: 画像処理完全移行（最優先）

**Pi側** - カメラ制御のみに特化:
```python
✅ 光検知（低解像度センサー利用）
✅ カメラ制御（ISO、SS、WB）
✅ RAW/JPEG撮影
✅ ファイル保存（無加工）
❌ 画像合成・加工は一切しない
```

**iPhone側** - 画像処理:
```swift
✅ ダウンロード後に合成
✅ タイムスタンプ追加
✅ サムネイル生成
✅ Core ImageでGPU加速
```

### Phase 2: 転送最適化

#### 現在の問題
```
Pi → iPhone: 12.3MP JPEG (10-15MB/枚)
多重露光: 2枚 × 15MB = 30MB転送
```

#### 改善案
```
Pi → iPhone:
- RAWモード: DNG (そのまま転送、後処理はiPhone)
- JPEGモード: quality=85 (8-10MB, 視覚的差なし)
- プレビューモード: 1920x1440 (2MB)
```

### Phase 3: 省電力撮影モード

```python
# 光検知間隔の動的調整
- 日中: 0.25秒 (現在)
- 夜間: 0.5秒 (負荷半減)
- 手動モード: カメラ完全停止
```

## 推定効果

### 消費電力削減
```
現在:
- アイドル: 0.4W
- 撮影+合成: 1.3W x 5秒 = 6.5Ws
- 1時間あたり: ~1.5Wh (撮影10回/h)

最適化後:
- アイドル: 0.4W
- 撮影のみ: 0.8W x 1秒 = 0.8Ws
- 1時間あたり: ~0.5Wh (66%削減)
```

### バッテリー寿命
```
5000mAh @ 5V = 25Wh
現在: ~16時間
最適化後: ~48時間 (3倍)
```
