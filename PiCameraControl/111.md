       接続監視はシングルトン ConnectionMonitor で実装されています。

       - 判定基準: HTTPステータスコード 200-499
       を「接続中」と判定（エラー応答でもPiが到達可能なら接続扱い）
       - URLSession設定: timeoutIntervalForRequest = 4秒, timeoutIntervalForResource = 6秒,
       waitsForConnectivity = false

       @MainActor
       final class ConnectionMonitor: ObservableObject {
           static let shared = ConnectionMonitor()
           @Published var isConnected: Bool = false
           @Published var lastChecked: Date?
           private var timer: Timer?
           private let checkInterval: TimeInterval = 5.0

           private init() {
               let config = URLSessionConfiguration.default
               config.timeoutIntervalForRequest = 4
               config.timeoutIntervalForResource = 6
               config.waitsForConnectivity = false
               self.session = URLSession(configuration: config)
           }
       }

       2. camera_service.py (カメラ制御)

       ファイル: /Users/sugawaraichirou/Documents/アプリ/home 3/camera_service.py (586行, 22KB)

       概要

       光検知ベースの自動撮影サービス。PiCamera2ライブラリを使用。

       光検知ロジック

       1. カメラメタデータからLux値を取得（取得不可の場合はloresストリームの平均輝度を使用）
       2. 前回の輝度からの変化率 (change_percent) と変化量 (change_amount) を計算
       3. 撮影条件:
         - change_percent >= threshold (デフォルト30%)
         - change_amount >= MIN_CHANGE_AMOUNT (5)
         - クールダウン期間経過
         - レートリミット未到達
         - 輝度の変化方向が正（明るくなる方向のみ）

       レートリミッター

       - 直近1分間の撮影回数で制限
       - モード別上限: reaction=24回/分、standard=15回/分、quality/night=10回/分、raw/battery=6回/分

       カメラ設定

       - Picamera2 ライブラリ使用
       - メインストリーム + lores(低解像度)ストリームの2ストリーム構成
       - loresストリームは光検知用（reaction: 128x96, 他: 160x120）
       - カメラ制御パラメータ:
         - AeEnable: ISO/シャッターがmanualの場合OFF
         - AnalogueGain: ISO値/100
         - ExposureTime: マイクロ秒
         - AwbMode: libcameraのAWBモード
         - NoiseReductionMode: denoise設定
         - Sharpness: 0.0-16.0

       画像保存ロジック

       - 保存先: /home/pi/photos/
       - ファイル名形式: photo_YYYYMMDD_HHMMSS_ffffff.jpg (自動撮影)
       - RAWモード時: photo_YYYYMMDD_HHMMSS_ffffff.dng
       - JPEG品質はプロファイルから取得

       センサーステータス書き込み

       - /home/pi/sensor_status.json にアトミック書き込み（tmp + rename）
       - 2秒間隔で更新
       - 記録内容: brightness, lux, ae_gain, ae_exposure_us, change_percent, capture_at等

       設定リロード

       - 2秒間隔で camera_settings.json + camera_session_overrides.json を再読み込み

       switch_to_ap_mode() の処理フロー

       1. hostapd, dnsmasq, dhcpcd を停止

       1. NetworkManagerのHotspot停止・削除
       2. NetworkManager停止・無効化
       3. wpa_supplicant再起動（per-iface confコピー含む）
       4. dhcpcd再起動
       5. wpa_cli reconfigure + reconnect
       6. 接続確認（最大30秒待機）
